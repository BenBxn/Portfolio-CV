Booki

Année : Mai 2022

Développement Web

Ce projet est le n°2 du parcours Développeur Web chez OpenClassRooms, qui consiste à transformer une maquette en site web avec HTML et CSS.

Informations
Contexte :

Stagiaire en tant que développeur web chez Booki, une petite entreprise proposant un outil de planification de vacances ! Son site permettra aux usagers de trouver des hébergements et des activités dans la ville de leur choix.
Chargé de créer un prototype en intégrant les maquettes réalisées par le designer UI, qui serviront de base de code pour le reste du développement, il faudra respecter les spécifications de design et les contraintes techniques reçues par mail via la CTO.

Cahier des charges :
Spécifications fonctionnelles & techniques :
  • Le texte du champ de recherche doit être éditable.
  • Les activités et hébergements doivent être cliquables en intégralité (liens vides).
  • Un effet de survol sur les filtres doit être présent.
  • Les liens "Hébergements" et "Activités" doivent mener à la section correspondante.
  • Nous disposons des maquettes Desktop et Mobile. Le site doit être également adapté sur tablette avec les modifications nécessaires.
  • Le travail consistait à travailler en desktop-first puis sur tablette puis sur téléphone.
  • Différentes tailles et formats d'image sont disponibles et il faut faire les meilleurs choix d'optimisation et de rendu.
  • Les icônes utilisées proviennent de Font Awesome.
  • La charte graphique est la suivante : `#0065FC` Bleu, `#DEEBFF` Bleu claire, `#F2F2F2` Gris.
  • Obligation d'utiliser les pixels et les pourcentages au lieu des REM et EM.
  • Obligation d'utiliser Flexbox uniquement (pas de Grid).
  • Interdiction d'utiliser un framework CSS ou préprocesseur CSS.
  • Utilisation des balises sémantiques en HTML.
  • Code valide au W3C : HTML & CSS
  • Compatible sur les dernières versions de Google Chrome et Mozilla Firefox.

OhMyFood!

Année : Juin 2022

Développement Web

Ce projet est le n°3 du parcours Développeur Web OpenClassRooms, qui consiste à dynamiser une page web avec des animations CSS.

Informations
Contexte :

Employé en tant que développeur junior chez Ohmyfood!, une jeune startup qui voudrait s'imposer sur le marché de la restauration. L'objectif est de développer un site 100% mobile qui répertorie les menus de restaurants gastronomiques. En plus des systèmes classiques de réservation, les clients pourront composer le menu de leur repas pour que les plats soient prêts à leur arrivée.
La commerciale chargée de démarcher les restaurants a réussi à convaincre 4 responsables de déposer leur carte en ligne. Un brief est établi en équipe avec le CTO. L'UX nous envoie les maquettes du site par mail.

Cahier des charges :
Spécifications fonctionnelles & techniques
  • Au survol les boutons principaux doivent légèrement s'éclaicrir avec une ombre plus visible.
  • Un bouton "J'aime" en forme de cœur doit se remplir progressivement en cas de clic ou de hover.
  • Un "loading spinner" cohérent avec la charte graphique doit s'afficher sur la page d'accueil.
  • Les groupes "Entrée", "Plat", "Dessert" doivent apparaître progressivement avec un léger décalage.
  • Sur un plat, une coche doit apparaitre en coulissant de la droite vers la gauche.
  • Nous disposons de la maquette mobile. Le site doit être également adapté sur tablette et desktop.
  • Le travail consistait à travailler en mobile-first puis sur tablette puis sur ordinateur.
  • Les icônes utilisées proviennent de Font Awesome.
  • La charte graphique est la suivante : Primaire #9356DC, Secondaire #FF79DA, Tertiaire #99E2D0.
  • Les polices doivent être pour le logo et les titres : Shrikhand, et pour le texte : Roboto.
  • Interdiction d'utiliser un framework CSS et/ou du JS.
  • Il est conseillé d'utiliser SASS.
  • Code valide au W3C : HTML & CSS
  • Compatible sur les dernières versions de Google Chrome et Mozilla Firefox.

Kasa

Année : Novembre 2022

Développement Web

Ce projet est le n°7 du parcours Développeur Web chez OpenClassRooms, qui consiste à Créer une application web de location immobilière avec React.

Informations
Contexte :

Kasa est dans le métier de la location d’appartements entre particuliers et recrute un développeur front-end en freelance pour développer sa nouvelle plateforme web.
la CTO lance une refonte totale pour passer à une stack complète en JavaScript avec NodeJS côté back-end, et React côté front-end.

Briefing :
  • Démarrer le projet React et développer l’ensemble de l’application, les composants React, les routes React Router, en suivant les maquettes Figma :
  • Maquette Figma - Kasa

  • Respecter les Outils et contraintes techniques:
  • Contraintes techniques - Kasa
Cahier des charges :
Outils et contraintes techniques :
  • Utiliser React et React Router.Css est à utiliser par défaut. Pas de librairie React externe.
  • Découper en composants modulaires et réutilisables.
  • Structure logique des fichiers.
  • Un composant par fichier.
  • Utilisation des props entre les composants.
  • Utilisation du state dans les composants quand c'est nécessaire.
  • Utiliser les listes en itérant dessus.
  • Gestion des événements.
  • Les paramètres des routes sont gérés par React Router dans l'URL pour récupérer les informations de chaque logement.
